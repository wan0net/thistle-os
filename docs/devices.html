ThistleOS / Supported Devices

Supported Devices

ThistleOS runs on the ESP32 chip family across 10 supported boards. One firmware binary per architecture. Hardware drivers are auto-detected by the Recovery OS via I2C/SPI/UART scanning and downloaded from the catalog.

One binary per architecture
The kernel is compiled separately for each architecture family: one binary for Xtensa (ESP32, S2, S3) and one for RISC-V (C3, C6, H2). Apps and drivers are also architecture-specific. The catalog arch field filters entries to the correct binary for each device.

How hardware detection works

When Recovery OS boots on a new or blank device, it auto-provisions hardware with no user configuration needed:

  1. Chip detection — Recovery reads the chip model register at boot. It knows whether it is running on ESP32, S2, S3, C3, C6, or H2, and selects the correct firmware architecture from the catalog.
  2. Power rail enable — Before scanning, Recovery enables peripheral power rails (where applicable) to ensure devices are powered.
  3. I2C bus scan — Probes addresses 0x08 through 0x77 on each I2C bus. Each responding address is looked up in the detection table. Known I2C addresses include: TCA8418 (0x34), CST328 (0x1A), BHI260AP (0x28), LTR-553 (0x23), PCF8563 (0x51), SSD1306 (0x3C/0x3D), and others.
  4. SPI device probing — GPIO-level CS pin checks determine whether SPI devices (e-paper, LoRa SX1262, SD card) are present by toggling CS and observing MISO behaviour.
  5. UART device probing — Listens on UART ports for GPS NMEA sentences (to detect MIA-M10Q or similar) and sends AT commands to detect cellular modems.
  6. Catalog match — Detected components are matched against catalog entries using the detection field. Matching driver .drv.elf files are selected for download.
  7. Bundle download — Recovery downloads the firmware binary (filtered by chip arch), matching drivers, and the appropriate window manager in one provisioning pass.
No board definition required
Recovery identifies hardware by component detection, not by board name. A new board with a TCA8418 keyboard and ST7789 display will automatically get the correct keyboard and LCD drivers without any manual configuration.

Adding your own board

To make a custom board work with ThistleOS:

  1. Create a board.json on SPIFFS with pin assignments, bus configs, and the driver list for your hardware. See Board Support for the JSON schema.
  2. Write (or reuse) driver manifests with detection fields — I2C addresses or SPI probe hints that Recovery can use to auto-detect your components.
  3. Add your board to the catalog with compatible_boards and arch fields so Recovery can match firmware and drivers to your device.
  4. Recovery auto-detects the hardware at first boot — no code changes to the kernel or core firmware needed.
