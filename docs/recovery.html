ThistleOS / Recovery OS

Recovery OS

A minimal Rust firmware in ota_0 that auto-detects hardware via I2C/SPI/UART scanning, downloads matching drivers and firmware from the catalog, and reinstalls ThistleOS without a programmer — via SD card, WiFi, or the UART console.

✎ Edit on GitHub

Purpose and design

The Recovery OS occupies the ota_0 partition — a 128 KB slot that the ESP-IDF bootloader can always fall back to. Its job is simple: make sure there is a working ThistleOS in ota_1. If there is not, it provides a way to install one.

Key design choices:

  • Rust + esp-rs — smaller binary than equivalent C code, and memory safety means fewer panic-class bugs in the one piece of code that must always work.
  • Hardware auto-detection — Recovery detects the chip variant (ESP32/S3/C3/etc.), then scans I2C buses (0x08–0x77), probes SPI device CS pins, and listens on UART for GPS NMEA / modem AT responses. Component matches are looked up in the catalog.
  • Multi-arch chip detection — a single Recovery binary identifies whether it is running on Xtensa or RISC-V and selects the correct firmware architecture from the catalog.
  • Full bundle download — Recovery downloads firmware binary, matching drivers, and the appropriate window manager in one provisioning flow. The device boots fully configured with no manual SD card setup.
  • No LVGL — the recovery UI is a captive portal web page served over WiFi, viewable on any phone or laptop. The e-paper display shows status messages via raw framebuffer writes.
  • No SD card required — if the SD card is missing or corrupt, recovery can still download firmware over WiFi.
  • UART console as fallback — if WiFi is not available, a serial console provides the same commands.

Captive portal web UI

When Recovery OS starts its WiFi AP, any device that connects to ThistleOS-Recovery (open network, no password) is automatically redirected to the captive portal. The portal is a single self-contained HTML page served from flash — no SD card needed.

The portal provides a 3-step guided flow:

  1. Step 1 — WiFi Connect — enter SSID and password to connect the ESP32 to your network (WPA2 client mode simultaneously with the AP). Recovery shows detected hardware components on this screen.
  2. Step 2 — Download & Install — fetches the firmware binary, matching drivers, and window manager from the official catalog URL (https://wan0net.github.io/thistle-apps/catalog.json), filtered to the detected chip architecture. Streams the bundle to flash and the SD card.
  3. Step 3 — Reboot — confirms installation; reboots the device into a fully provisioned ThistleOS.

Captive portal redirect paths
iOS, Android, and Windows all use different URLs to detect captive portals. The Recovery OS handles /generate_204 (Android), /hotspot-detect.html (iOS), and /connecttest.txt (Windows), redirecting all of them to /.

UART console commands

Connect at 115200 baud, 8N1. Hit Enter after each command.

CommandDescription
helpShow available commands
statusRecovery version, WiFi state, ota_1 state, SD firmware presence
scanScan for WiFi networks and list up to 15 results with RSSI and channel
connect SSID,PASSConnect to a WiFi network (WPA2). No spaces in SSID or password.
downloadDownload and install ThistleOS from the app store catalog (requires WiFi)
rebootRestart the device

REST API endpoints

The captive portal web server also exposes a small REST API for scripted recovery:

MethodPathDescription
GET/Captive portal HTML page
GET/api/statusJSON: version, mode, ota1 state, sd_firmware
POST/api/wifi/connectBody: {"ssid":"...","password":"..."}. Returns {"ok":true,"ip":"..."}
POST/api/ota/downloadTriggers firmware download from catalog. Returns {"ok":true} then reboots.
POST/api/rebootReboots the device after 1 second.

Building from source

The Recovery OS is a standard esp-idf-svc Rust project. You need:

  • Rust toolchain with the xtensa-esp32s3-espidf target (and/or RISC-V target for C3/C6/H2)
  • espup (Espressif's Rust toolchain installer)
# Install espup and the Xtensa Rust toolchain
cargo install espup
espup install

# Activate the toolchain in your shell
. $HOME/export-esp.sh

# Build the recovery firmware
cd recovery
cargo build --release

# Flash to ota_0 partition specifically
espflash flash --partition-table partitions.csv \
    --partition ota_0 \
    target/xtensa-esp32s3-espidf/release/thistleos-recovery
Toolchain
The Recovery OS uses esp-idf-svc which builds a vendored ESP-IDF internally. The first build takes several minutes and requires internet access to download the ESP-IDF toolchain managed by .embuild/.
