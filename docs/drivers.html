Driver Reference

ThistleOS uses a HAL vtable driver model: every hardware peripheral is accessed through a C struct of function pointers. Drivers implement these vtables in Rust (17 drivers) or C (2 drivers), register them with the HAL registry at boot, and the kernel dispatches calls without knowing which chip is behind the interface.

Status legend

Proven Tested on real hardware, no known issues. Fixed Had bugs found in review, now corrected and aligned with Espressif references. Partial Some functionality implemented but not fully tested. Stub Placeholder; hardware calls are no-ops.

Implementation patterns

  • Rust wrapping ESP-IDF — The Rust driver calls an official ESP-IDF C API via FFI (e.g. esp_lcd_new_panel_st7789()). Safest path when Espressif provides a mature API.
  • Rust, manual I2C/SPI — The Rust driver directly reads and writes registers over the bus via i2c_master_transmit() / spi_device_transmit(). Used when no ESP-IDF component exists for the chip.
  • C, wraps library — A C driver wrapping an external library (RadioLib, esp_modem). Used for complex protocols where a mature C/C++ library already exists.

Display drivers

drv_lcd_st7789 Proven

SPI LCD driver for the ST7789V controller. Uses the ESP-IDF esp_lcd component for panel initialization and bitmap flushing. Supports configurable resolution, pixel offsets, and color inversion for different panel modules.

HAL TypeDisplay
ChipST7789V
InterfaceSPI
Sourcecomponents/kernel_rs/src/drv_lcd_st7789.rs (918 lines)
WrapsESP-IDF esp_lcd_new_panel_st7789(), esp_lcd_panel_draw_bitmap()
Tests20 unit tests
BoardsT-Deck, T-Deck Plus, T-Display-S3, Cardputer
NotesConfigurable width/height/offset. Cardputer uses 240x135 with x_offset=40, y_offset=52.

drv_lcd_ili9341 Fixed

SPI LCD driver for the ILI9341 controller. Uses manual initialization via esp_lcd_panel_io_tx_param() with a ST7789 panel handle, since ILI9341 shares the same draw-bitmap interface but needs a different init sequence.

HAL TypeDisplay
ChipILI9341
InterfaceSPI
Sourcecomponents/kernel_rs/src/drv_lcd_ili9341.rs (1117 lines)
WrapsESP-IDF esp_lcd_panel_io_tx_param(), uses ST7789 panel handle
Tests23 unit tests
BoardsCYD S028
NotesFixed: manual init sequence aligned with ILI9341 datasheet. Shares draw path with ST7789 driver.

drv_oled_ssd1306 Proven

I2C OLED driver for the SSD1306 controller. Sends raw I2C commands for initialization, page addressing, and framebuffer flushing. Supports 128x64 and 128x32 panels.

HAL TypeDisplay
ChipSSD1306
InterfaceI2C
Sourcecomponents/kernel_rs/src/drv_oled_ssd1306.rs (871 lines)

Tests22 unit tests
BoardsHeltec V3, T3-S3, C3-Mini
NotesMonochrome 1-bit display. Kernel converts RGB565 framebuffer data to 1-bit pages before flushing.

drv_epaper_gdeq031t10 Proven

SPI e-paper driver for the GDEQ031T10 (Good Display 3.1" 4-gray). Manual SPI command sequences for full refresh, partial refresh, and fast mode. Integrated with the kernel's epaper_refresh module for coalesced updates.

ADC-based battery monitor for LiPo cells charged by the TP4065B. Reads raw ADC voltage via the ESP-IDF oneshot ADC API and maps it to a percentage using a LiPo discharge curve.

HAL TypePower
ChipTP4065B
InterfaceADC
Sourcecomponents/kernel_rs/src/drv_power_tp4065b.rs (967 lines)
